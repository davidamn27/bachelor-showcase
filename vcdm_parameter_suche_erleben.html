Use Case Parametersuche

03 Anwendung & Lösung

Projekt nach einem Parameter durchsuchen

Der Use Case zeigt, wie in vCDM ein Projekt geöffnet, der Analysebereich genutzt und ein bestimmter Parameter über mehrere Datensätze hinweg gefunden und verglichen werden kann.

01 Projekt öffnen

Projektkontext herstellen

  • Fahrzeug- oder Projektstruktur öffnen
  • Relevanten Datenstand auswählen
  • Ausgangspunkt für die Suche festlegen
02 Analyse starten

Analysebereich nutzen

  • In Analyse und Reporting wechseln
  • Value Population Matrix öffnen
  • Parametervergleich vorbereiten
03 Datensätze wählen

Suchraum eingrenzen

  • Relevante Datensätze markieren
  • Softwarestände einbeziehen
  • Vergleichsbasis definieren
04 Parameter Scope

Parameterfilter aufbauen

  • Parameterbereich öffnen
  • Gezielte Auswahl vorbereiten
  • Unnötige Treffer reduzieren
05 Parameter auswählen

Gesuchten Parameter finden

  • Parameterliste durchsuchen
  • Passende Einträge auswählen
  • Auswahl bestätigen
06 Treffer übernehmen

Auswahl prüfen

  • Gefundene Parameter kontrollieren
  • Filterliste übernehmen
  • Analyse fortsetzen
07 Einstellungen

Vergleich konfigurieren

  • Quantisierung bewusst wählen
  • Vergleichslogik festlegen
  • Auswertung abschließen
08 Zusatzdaten

Informationen ergänzen

  • Weitere Attribute einblenden
  • Management- oder A2L-Informationen nutzen
  • Kontext für Bewertung erweitern
09 Ergebnis

Parameterwerte sehen

  • Treffer in Matrix anzeigen
  • Werte über Datensätze vergleichen
  • Abweichungen schnell erkennen
10 Bewertung

Projektweit vergleichen

  • Mehrere Parameter zusammen auswerten
  • Variantenunterschiede sichtbar machen
  • Entscheidung oder Anpassung vorbereiten
