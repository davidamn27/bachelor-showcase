Use Case Variantenerstellung

03 Anwendung & Lösung

Neue Variante durch Parameteränderung erstellen

Der Use Case zeigt, wie aus einem bestehenden Datenstand eine neue Variante entsteht: Parameter auswählen, Wert ändern, Änderung nachvollziehen und die neue Variante im Projektkontext sichern.

01 Ausgangspunkt

Datenstand auswählen

  • Projektstruktur öffnen
  • Bestehenden Datensatz markieren
  • Basis für neue Variante festlegen
02 Bearbeitung

Datensatz öffnen

  • Relevante Parameteransicht starten
  • Bearbeitbaren Stand herstellen
  • Änderung gezielt vorbereiten
03 Parameter finden

Parameter auswählen

  • Gewünschten Parameter suchen
  • Passenden Eintrag markieren
  • Änderungsstelle eindeutig festlegen
04 Wert ändern

Neue Ausprägung setzen

  • Parameterwert anpassen
  • Änderung im Datensatz sichtbar machen
  • Variante fachlich ableiten
05 Übernehmen

Änderung bestätigen

  • Anpassung prüfen
  • Änderung übernehmen
  • Bearbeitungsstand erzeugen
06 Variante

Neuen Stand einordnen

  • Geänderten Datensatz sichtbar machen
  • Neue Variante im Projektkontext prüfen
  • Unterschied zum Ausgangsstand erkennen
07 Nachvollziehbarkeit

Änderung dokumentieren

  • Metadaten und Zustand prüfen
  • Änderung historisierbar machen
  • Verantwortlichkeit nachvollziehen
08 Vergleich

Variante prüfen

  • Neue und alte Werte gegenüberstellen
  • Abweichung fachlich bewerten
  • Freigabefähigkeit vorbereiten
09 Ergebnis

Neue Variante verfügbar

  • Parameteränderung ist im Projekt abgelegt
  • Variante bleibt nachvollziehbar
  • Weitergabe oder Freigabe kann folgen
